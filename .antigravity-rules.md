# iStorePro — Regras de Desenvolvimento (Toque Mínimo)

> **LEIA ANTES DE QUALQUER ALTERAÇÃO.**  
> Estas regras garantem que mudanças cirúrgicas não quebrem outras áreas do sistema.

---

## 1. Princípio do Toque Mínimo

- ✅ **Altere APENAS o que foi pedido.**  
- ❌ Nunca refatore, renomeie, reordene ou "melhore" código fora do escopo da tarefa.
- ❌ Nunca troque um bloco funcional por outro apenas por preferência de estilo.
- Se perceber um bug em outra área durante uma tarefa, **documente-o em um comentário `// BUG: ...` e reporte ao usuário** — mas NÃO corrija sem autorização.

---

## 2. Regras de Serviços (`services/`)

### `productService.ts` — `updateProduct()`
- O payload enviado ao Supabase `.update()` NUNCA deve conter campos que não existam como colunas reais na tabela `products`.
- **Campos proibidos no updatePayload** (devem ser desestruturados e descartados antes do spread):
  - `relevance_score` — campo virtual retornado pelo RPC `search_products`
  - `total_count` — campo virtual retornado pelo RPC `search_products`
  - `totalCostPrice` — campo computado em memória (`costPrice + additionalCostPrice`), não existe no banco
  - `selectedCustomerId` — campo de contexto do formulário, não é coluna do banco
  - `stockHistory` — só pode ser atualizado via funções específicas (ex: `updateProductStock`)
  - `priceHistory` — gerenciado internamente após comparação de preços
- **Ao adicionar qualquer novo campo ao objeto `Product`**, verifique se é uma coluna real no Supabase. Se for apenas computado/derivado, adicione-o à lista de exclusão acima.

### `searchProductsRPC()` / resultados de RPC
- Ao mapear resultados de RPC para objetos `Product`, campos extras do RPC (como `relevance_score`, `total_count`) NÃO devem ser adicionados ao tipo `Product`.
- Se necessário temporariamente, use um tipo local `ProductWithRPCMeta` para não contaminar o objeto canônico.

---

## 3. Regras de Componentes (`components/`, `pages/`)

- Alterações em um componente NÃO devem afetar outros componentes, mesmo que compartilhem CSS ou contexto.
- Mudanças de layout (ex: classes do Tailwind) devem ser mínimas — adicione classes, não substitua blocos inteiros.
- Ao corrigir overflow/layout em um modal, verifique se a correção se aplica **apenas** ao modal alvo. Use classes específicas ao container afetado.

---

## 4. Regras de Banco de Dados / Supabase

- **Nunca tente `update()` com campos virtuais** — sempre desestruture e descarte campos não-coluna.
- Sempre que uma função RPC retornar dados extras, mapeie para `Product` explicitamente (campo por campo), não com spread genérico `...p`.
- Qualquer novo campo que possa vir de RPC deve ser testado contra o schema antes de ser incluído em queries de escrita.

---

## 5. Commits e Deploy

- Cada commit deve corresponder a **uma única correção ou funcionalidade**.
- Mensagem de commit no formato: `fix(módulo): descrição curta` ou `feat(módulo): descrição curta`.
- Antes de subir para o GitHub (e acionar o deploy na Vercel), teste localmente com `npm run dev`.

---

## 6. Campos Computados vs. Colunas Reais — Referência Rápida

| Campo JS          | É coluna no Supabase? | Observação                                  |
|-------------------|-----------------------|---------------------------------------------|
| `totalCostPrice`  | ❌ Não                | Calculado: `costPrice + additionalCostPrice` |
| `relevance_score` | ❌ Não                | Retornado pelo RPC `search_products`         |
| `total_count`     | ❌ Não                | Retornado pelo RPC `search_products`         |
| `costPrice`       | ✅ Sim                | Coluna real                                  |
| `wholesalePrice`  | ✅ Sim                | Coluna real                                  |
| `stockHistory`    | ✅ Sim (JSONB)        | Atualizar apenas via funções específicas     |
| `priceHistory`    | ✅ Sim (JSONB)        | Atualizar apenas via `updateProduct`         |
| `supplierId`      | ⚠️  Alias            | Coluna real é `supplier_id` (snake_case)     |
| `storageLocation` | ⚠️  Alias            | Coluna real é `storage_location`             |
| `serialNumber`    | ⚠️  Alias            | Coluna real é `serial_number`                |
| `batteryHealth`   | ⚠️  Alias            | Coluna real é `battery_health`               |
| `createdAt`       | ⚠️  Alias            | Coluna real é `created_at`                   |
| `updatedAt`       | ⚠️  Alias            | Coluna real é `updated_at`                   |
